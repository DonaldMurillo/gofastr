# UI-quality evaluation results

## Current framework evidence

The causal protocol is now correct: a neutral product brief goes to an OMP /
GLM-5.2 builder using only the ordinary generated GoFastr scaffold, docs, and
host skill. Independent Claude Code / Opus agents judge anonymized rendered
screenshots. No evaluator design override is supplied and evaluated candidates
are never edited in place.

The framework has repeatedly transferred its adaptive theme, mobile identity,
linked chrome, `RecordSummary`, `MetricBand`, compact support rail, early
primary action, and zero-app-CSS contract into fresh neutral candidates. The
best valid single-scenario samples reached **8.255 holistic / 8.595 mobile** and
**8.185 holistic / 8.645 mobile**, both with technical passes and shadcn-level
consensus. This is a material gain over the first corrected-protocol sample at
7.58 / 7.46.

The latest diagnostic sequence also found framework defects that are now
regression-tested upstream:

- readable AvatarGroup overlap, adaptive overflow chip, and compact presence
  dots;
- balanced odd-count phone MetricBands;
- intact button labels combined with action rows that wrap whole controls;
- `ClusterConfig{}` now truly wraps by default, with `NoWrap` as the explicit
  compact-chrome opt-out;
- wide detail-route guidance that pairs related bounded modules instead of
  leaving an accidental empty desktop rail;
- bounded OMP implementation passes plus Windows job and Unix process-group
  ownership, covering both timeout and normal-exit descendant cleanup;
- fail-closed framework-input checks after every builder and candidate, so an
  unrestricted agent cannot contaminate later matrix cells.

The final framework snapshot passes its focused desktop/mobile light/dark
rendered contract, the full component-gallery axe sweep, and Meridian's
marketing/auth/app/admin desktop/mobile light/dark visual and axe canary.

These runs are diagnostics, not a promotion certificate: each selected one of
five scenarios and one of three required judges per panel. A release-quality
claim still requires the configured multi-scenario, multi-judge, multi-snapshot
matrix. A single sample is intentionally not allowed to name a winner.

## Invalidated prompt-treatment history

Runs named `baseline`, `composition-v1` through `composition-v5`, including
`composition-v5-hardened-final-20260713-01`, do **not** answer whether GoFastr
naturally produces strong UI. They used evaluator-supplied design overrides,
disabled normal repository rules, or allowed large app-owned stylesheets. Their
scores remain historical diagnostic references only and are ineligible for
framework claims.
