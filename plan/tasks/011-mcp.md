# 011 — MCP Primitive

**Phase:** 1 (Core Primitives) | **Tier:** 2 | **Depends on:** 004, 005

## Goal
Model Context Protocol server. Register tools, handle calls via JSON-RPC 2.0 over stdio and HTTP/SSE.

## Deliverables
- [ ] `MCPServer` struct with tool registry
- [ ] `Tool` definition: Name, Description, InputSchema (JSON Schema), Handler function
- [ ] Tool handler reuses Handler primitive (typed in/out)
- [ ] JSON-RPC 2.0 message handling: parse request, route to handler, return response
- [ ] Transport: stdio (stdin/stdout), HTTP+SSE (POST for calls, SSE for streaming)
- [ ] `tools/list` handler — return all registered tools
- [ ] `tools/call` handler — execute tool with params, return result
- [ ] Auto-generate input schema from Go struct tags (via Schema primitive)
- [ ] Context propagation: auth user, tenant info passed to tool handlers
- [ ] Error responses per MCP spec (invalid params, tool not found, execution error)

## Acceptance Criteria
- `tools/list` returns valid MCP JSON
- `tools/call` executes typed handler, returns result
- Both stdio and HTTP transports work
- Invalid tool name returns proper MCP error
- Invalid params return proper MCP error
