# Impeccable design context for GoFastr

(Distilled from the 2026-06 website design brief + the v2 design chat —
the brief itself now lives in git history. Kept current so /impeccable
invocations skip the teach step.)

## Design Context

### Users
Primary: mid-to-senior Go backend engineers who already pair with AI
coding agents (Claude / Cursor / Copilot) and want a Go full-stack
framework that meets the agent halfway. Comfortable in `database/sql`,
`net/http`, channels. Skeptical of magic; will read the generated code
before trusting it. Allergic to client-side complexity; notice that
this site uses no React.

Secondary: agentic-coding enthusiasts curious about Kiln-style "describe
an app, watch it build."

Tertiary: Go community looking for a modern framework.

NOT for: enterprise procurement (pre-alpha), no-code visitors, total
beginners learning to program.

### Brand Personality
**Three words**: pre-alpha honest · technically dense · engineer voice.

Tone signals (from existing source):
- Commit messages and docs name the bug, name the cause, name the
  fix. Direct, ego-free, named-bug specificity.
- "Declare once, generate many surfaces." "No reflection magic. Read
  the generated code." "Drop down to core."
- Pro-AI without hype. "The agent and the engineer build the app
  together and the framework respects both."
- Strong opinions, small scope. Says no to things.

### Aesthetic Direction
**Dark-first**. Warm-near-black surface ladder in oklch; off-white
text; **amber** as the single chromatic accent (oklch 0.82 0.155 78).
NOT the red/blue every dev tool picks. Sans + mono only — no serif
(magazine direction was rejected in the v1 → v2 pivot).

Currently using system stack (Geist substituted with `-apple-system,
BlinkMacSystemFont, Inter, system-ui, sans-serif`; mono falls through
to `ui-monospace, SFMono-Regular, JetBrains Mono`). Open to swapping
to a more distinctive face when the project is ready to load web fonts
under a loosened CSP.

**Anti-references** (do not converge to):
- AI-startup gradients, glassmorphism, neon-on-dark
- "Generate X with Y in seconds!" hero copy
- Webflow-template SaaS layouts (logo cloud, pricing tiers, testimonials)
- Heavy WebGL hero scenes
- Cliché terminal mockups with `npm install` when the actual command
  is `go install ./cmd/gofastr`
- Magazine cosplay — uppercase mono nav, `//` decorations, slogan
  headlines. That was the rejected v1.

**Yes-references**: Linear, Astro, Tailscale, Cloudflare docs, Mitchell
Hashimoto's blog — restraint + technical writing as marketing.

### Design Principles
1. **Code is the hero**. Code samples carry the explanation, not slogans.
2. **Restraint wins ties**. The audience reads for a living; reward
   them with depth, but don't overwrite or oversell.
3. **Pre-alpha honest, not pre-alpha apologetic**. Surface the status
   plainly; don't hide it, don't overdramatize it.
4. **Dogfood the framework**. Every interactive element on this site
   is a registered component shipped by GoFastr itself. If a pattern
   needs >20 lines of site-local glue it belongs as a framework
   primitive instead.
5. **Strict CSP, server-rendered, hydrated**. No inline `<script>`/
   `<style>`. JS is enhancement, not foundation. First paint is real
   HTML. Cross-page nav is client-side with cache; in-page state is
   islands; server-pushed updates flow through SSE.
6. **Accessible, mobile-first, theme-tokenized**. WCAG AA contrast,
   keyboard-navigable, screen-reader announceable. Works at 320px.
   Uses the framework's typed theme so light/dark switches cleanly.

### Technical Constraints
- Go full-stack framework (GoFastr). UI built via `framework/ui` +
  `core-ui/html` + `core-ui/patterns` + `core-ui/widget`. CSS through
  the typed `style.NewStyleSheet(theme)` DSL.
- Strict CSP (`default-src 'self'`). No inline styles. No external
  fonts unless we open `font-src` (currently closed).
- Site is `examples/site/` (the single canonical example app).
- Dev-watch loop rebuilds + reloads on save (TARGET=examples/site
  PORT=8083 ./scripts/dev-watch.sh).
